Astrolabe User Manual

A comprehensive guide to mapping your Crystal Spheres.

1. Menu Items

The top application menu provides global controls for file management and application settings.

Menu Item Shortcut Description
File > New Sphere Cmd/Ctrl + N Creates a brand new, blank Crystal Sphere configuration with default starting values.
File > Open Sphere... Cmd/Ctrl + O Opens the Directory Pane on the left, allowing you to browse and load an existing system save file from your saves/ directory.
File > Save Cmd/Ctrl + S Saves the current state of the system directly to its active JSON file.
File > Save As... Cmd/Ctrl + Shift + S Prompts you to specify a new filename to duplicate or save your current system state.
Edit > (Undo/Cut/Copy) Standard Standard text-editing shortcuts for interacting with input fields in the System Editor.
Help > Documentation - Opens this User Manual.
Help > About Astrolabe - Shows application version information.

2. System Editor Overview

The System Editor (the left-hand sidebar) is where you configure the physical traits, orbits, and metadata of your star system. It consists of two main sections: System Config and Celestial Bodies.

System Config

  • Sphere Name: The global name of the system (e.g., "Realmspace").
  • System Time (Epoch, Year, Day): Sets the baseline "Time 0" calendar date for the system. This is the default date the system resets to.
  • Shell Boundary: Configures how the outermost crystal shell is calculated (Double distance or Custom multiplier).
  • Orbital Line Strength: Adjusts the opacity/thickness of the dashed orbital rings on the map.
  • Title Strike Outline: Adds a legible outline behind text labels to ensure they are readable against busy map backgrounds.

Using the Editor Tools

  • Adding Objects: Click the Add Object button to create a new Celestial Body, Phenomenon, Constellation, Overlay Note, or Group folder.
  • Drag & Drop: Click and hold the header of any object card to drag it up or down the list. Dropping an object onto a Group folder will nest it inside. Dropping it in the dashed box at the bottom moves it back to the root level.
  • Visibility Toggle (👁️): The eye icon on the card header temporarily hides the object from the map canvases.
  • Delete (🗑️): Removes the object entirely. Any objects orbiting a deleted parent will be reset to orbit the center of the system.
  • Affects Shell Boundary: If unchecked, the object's distance is ignored when calculating the size of the outer crystal shell. (Useful for distant comets or background elements).
  • DM Only: Hides this object (and its children, if it's a group) when the global view mode is set to "Player View".

3. Object Types & Configuration

Every object in Astrolabe has unique configuration parameters based on its type.

Physical Bodies Star, Planet, Moon, Asteroid, Station, Custom

Standard solid objects that occupy physical space and orbit a parent body.

  • Size Class & Physical Size: Sets the physical diameter (Size A through J).
  • Orbit Parent: Select which other body this object orbits. If left blank, it orbits the center (0,0). Moons automatically orbit Planets.
  • Distance (AU): The radius of the orbit.
  • Initial Angle: Where the object starts on its orbital ring (0-360 degrees) at Time 0.
  • Orbital Period: Determines how fast the object moves. If set to Auto, it is derived mathematically from the distance.
  • Motion Mode:
    • Prograde: Standard counter-clockwise orbit.
    • Fixed: Stationary object; ignores time progression.
    • Retrograde: Clockwise orbit.
  • World Shape: Choose the visual representation (Sphere, Disc, Cluster, etc). Stations gain access to artificial shapes (Ring, Castle, Skull, Ship).
  • Elemental Affinity: Applies a color-coded alchemical badge to the object (Fire, Water, Earth, Air).

Note: Living Worlds feature additional sliders to generate procedural branching tree structures (Levels, Density, Leaves).

Phenomena Cloud, Nebula, Sargasso

Volumetric environmental hazards or features that smear along an orbital path.

  • Arc Degrees: How wide the cloud stretches along its orbital ring (1-360 degrees).
  • Transparency & Cloudiness: Controls the visual opacity and edge-bumpiness of the phenomenon.
  • Internal Objects: You can configure the shape, size, and density of particulate matter (e.g., asteroids, debris) drawn inside the cloud volume.

Constellations Background Stars

Decorative star-patterns drawn in the background space.

  • Shape: Choose from classic silhouettes or class-based icons (Aries, Dragon, Wizard, etc).
  • Level of Detail (LOD): Controls the complexity of the connecting wireframe lines (1 = simple, 20 = highly detailed).
  • Star Count: How many individual stars to randomly scatter across the wireframe nodes.
  • Style: Choose whether the constellation draws an "Internal" skeletal wireframe, or a perimeter "Outline".
  • Elemental Affinity: Setting this to "Mixed" will randomly assign vivid colors to the individual stars.

Map Overlays Note, Legend

Purely visual typography or graphical legend boxes.

  • Text Note: Add custom text to the map. Notes support custom Google Fonts, rotation, font sizes, and text wrapping. On the Navigation Chart, you can click and drag the 4 corners of a Note to create custom skewed text boxes.
  • Legend: Automatically generates a graphical key for Planet Types, Orbit Types, or Elemental Affinities currently present in the system.

Groups Folder

Groups do not draw anything on the map themselves, but act as organizational folders.

  • Inheritance: Toggling the DM Only or Visibility state of a Group automatically applies that state to everything inside it.
  • Nesting: Groups cannot be nested inside other Groups.
